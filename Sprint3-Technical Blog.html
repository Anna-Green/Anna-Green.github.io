CSS and HTML



HTML is like the skeleton of the body. It performs the role of adding structure and is what everything else is built upon.
You can see the HTML but its not necessarily the best thing to look at. CSS is the skin, hair, eyes piercings etc.
It is all placed upon the skeleton but you can change up the hair colour, add as many tattoos or piercings as you want and even change your eye colour with contact lenses.
CSS is what makes the website fun to look at.



Control Flow and Loops

Control flow is the way in which a code is run. Just like cooking. You read the predetermined recipe (instructions) going in the correct order in order to successfully complete your task.
One element that can have an affect on control flow are somethings called loops. Just like it sounds like, a loop loops your specified instruction so essentially you will be cooking for as long as the code tells you to.
Loops allow you to run the code for as long as you would like it to or until it has met a certain condition.



The DOM



The DOM stands for Document Object Model.
It is a user interface that allows you to change your webpage without changing the actual HTML code.
It is essential for developers who want to alter a site. It reads the document as a tree with branches of elements, attributes and nodes that contain the HTML code.



Arrays and Objects



Arrays and objects are both ways that we collect and contain data.
So why do we have two and which one is better to use?

Arrays:

Arrays are good if you have lots of single data and you want to put them into a collection.
Its also good if you need your data sorted ie: alphabetically, numbered etc…
Var sandwich = [“Lettuce”, “Tomato”, “Bread”]

Objects:

Objects are good if you need to contain multiple pieces of data which are then broken down further into more data.
For example:
Var cat = {
“name”: “Whiskers” ,
“legs”: 4,
“tails: 1,
“enemies”: [“water”, “dogs”]
the variable cat being the object, you can store further information into the properties of how many legs the cat has etc…
So there is not one that is better to use, it just depends on your data and what your intention is.



Functions



A function is a JavaScript code for performing a task (function).
It has pre-set instructions and once you have declared the function, all you must do is call it.
You can delete, replace or do whole lot more with functions including creating and declaring your own.
It can be tremendously helpful and acts as shortcuts throughout your code.
